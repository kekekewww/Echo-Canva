# Build Week Submission Checklist

## Project eligibility and evidence

- [ ] Project work is documented within the official submission period.
- [ ] Dated commits are preserved.
- [ ] Core work occurred in one identifiable Codex project thread.
- [ ] `/feedback` Codex Session ID is captured.
- [ ] Third-party code and assets have compatible licenses.
- [ ] Repository includes a license.
- [ ] No pre-existing work is presented as new without clear distinction.

## Working project

- [ ] Public deployment is available without payment or special hardware.
- [ ] Preset demo works even if GPT calls fail.
- [ ] AI scene compiler works with GPT-5.6.
- [ ] Raw/Simulated A/B works.
- [ ] Occlusion wow moment works.
- [ ] Portal-direction wow moment works.
- [ ] Import/export works.
- [ ] No critical console error.
- [ ] `pnpm verify` passes.
- [ ] Production smoke test passes.

## Repository

- [ ] README is in English.
- [ ] README contains product problem and audience.
- [ ] README contains architecture and data flow.
- [ ] README contains setup and environment instructions.
- [ ] README contains supported browser/platform.
- [ ] README contains test commands.
- [ ] README contains a direct demo URL.
- [ ] README explains where Codex accelerated work.
- [ ] README explains key human decisions.
- [ ] README explains how GPT-5.6 is used at runtime.
- [ ] README lists limitations and non-goals.
- [ ] README lists asset licenses.
- [ ] `.env.example` exists.
- [ ] No `.env` or credential is tracked.

## Demo video

- [ ] Public YouTube link.
- [ ] Under three minutes.
- [ ] English narration, captions, or full English translation.
- [ ] Shows the product working.
- [ ] Shows natural-language scene compilation.
- [ ] Shows wall occlusion.
- [ ] Shows portal re-panning.
- [ ] Shows Raw/Simulated A/B.
- [ ] Shows deterministic diagnostics.
- [ ] Explains Codex use.
- [ ] Explains GPT-5.6 use.
- [ ] Uses only licensed audio and visuals.

## Suggested shot list

1. 0:00–0:15 — Problem and one-sentence value.
2. 0:15–0:40 — Natural-language scene to editable canvas.
3. 0:40–1:10 — Drag behind wall; hear occlusion.
4. 1:10–1:35 — Open doorway; hear portal direction.
5. 1:35–1:55 — Concrete versus soft room.
6. 1:55–2:15 — Raw/Simulated A/B and diagnostics.
7. 2:15–2:35 — Export and explanation.
8. 2:35–2:55 — Codex/GPT-5.6 architecture and impact.

## Final consistency check

- [ ] Deployment commit equals release tag.
- [ ] Screenshots match current UI.
- [ ] Video matches current behavior.
- [ ] README commands work.
- [ ] Devpost description makes no unsupported accuracy claims.
- [ ] Product is described as an acoustic approximation/previsualization tool.
