# Echo Canvas Gate A Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver the Gate A candidate: a tested Next.js application with validated preset scenes, an editable 2D floor plan, and persistent raw/browser-HRTF audio playback.

**Architecture:** Keep domain logic framework-independent, put editor mutations behind a reducer, and put Web Audio behind an `AudioEngine` boundary. Create stable source graphs only after an explicit user gesture; later acoustic frames will update these graphs without rebuilding them.

**Tech Stack:** Next.js App Router, React 19, TypeScript strict, Zod, Vitest, Testing Library, Playwright, Web Audio API, pnpm.

## Global Constraints

- Follow `AGENTS.md`; stop after Human Gate A.
- Use the exact `SceneSpec` contract in `docs/API_CONTRACTS.md` and `schemas/scene-spec.schema.json`.
- Enforce 100 walls, 8 portals, 4 point sources, 6 early-reflection taps, 50 m coordinates, 0.1 m wall length, and 2,000-character prompts.
- Use only built-in material and audio IDs; no remote audio URLs.
- Audio begins only after an explicit Start Audio action.
- `PannerNode.panningModel` is `"HRTF"` and `rolloffFactor` is `0`; distance attenuation is manual.
- Audio parameters use smoothing and source graphs are not rebuilt during scene edits.
- Use product language from `AGENTS.md`; do not make excluded claims.
- Use test-driven development for behavior changes. Configuration and static presentation are covered by lint, typecheck, build, and e2e.
